Class: telemetryService

telemetryService

new telemetryService()

Telemetry service helps to log telemetry events. Telemetry service generates below listed events and logs to registered dispatchers.
  1. V2 => V3
  2. CE_START => START
  3. CE_API_CALL => LOG
  4. CE_INTERACT => INTERACT
  5. CE_PLUGIN_LIFECYCLE => INTERACT
  6. CE_ERROR => ERROR
  7. CE_END => END

(static) initialize(context, dispatcher)

Deperecated: V3 telemetry implementation Initialize the service with context and dispatcher.
Parameters:
Name Type Description
context object context object can have uid, sid, context_id. context should have content id, otherwise telemetry service cannot be initialized.
dispatcher string There are 3 types of dispatcher available, anyone of dispatcher is allowed and same dispatcher is used throughout the editor session.
  1. Local dispatcher - dev environment only, logs to file (telemetry.log) - value: "local"
  2. piwik dispatcher - logs to piwik endpoint - value: "piwik"
  3. console dispatcher - logs to client console, default dispatcher - value: undefined
